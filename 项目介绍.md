# AirPhone 暖通控制板监控项目 - 完整介绍

## 项目概览

这是一个 **暖通空调（HVAC）控制板的监控和调试系统**，由两个部分组成：

| 部分 | 功能 | 技术栈 |
|------|------|--------|
| **固件** | 运行在控制板 MCU 上的嵌入式程序，接收命令、控制硬件、上报状态 | C语言，Renesas RL78/G13 (R5F100LG)，CC-RL 编译器 |
| **上位机** | Windows 桌面监控软件，通过串口与 MCU 通信，实时显示状态、发送控制命令 | Python 3, PySide6, pyserial, pyqtgraph |

两者通过 **RS-485 串口（9600 8N1）** 连接，支持两种通信协议并存：
- **AA55 自定义二进制协议**：帧头 `AA 55`，长度+地址+序号+命令+控制+数据+CRC，最多 32 字节数据区
- **Modbus RTU 标准协议**：功能码 0x01/0x02/0x03/0x04/0x05/0x06/0x08

---

## 硬件信息

- **MCU**：Renesas R5F100LG（RL78/G13 系列）
- **调试 LED**：P77（P7_bit.no7），低电平点亮，高电平熄灭
- **RS-485 硬件 UART**：UART1，TX=P02, RX=P03, 方向控制=P27（协议中定义为 `P77` 脚位）
- **软件 UART**：通过定时器位拆实现，TX=P01, RX=P00（备用通道，当前未使用）
- **程序入口**：
  - 启动文件 `cstart.asm` → `_hdwinit` → `R_Systeminit()`（初始化所有外设）
  - → `main()` 在 `r_main.c` 中 → `R_MAIN_UserInit()`（初始化 RS485 + 协议）→ 主循环

---

## 通信架构（固件端）

固件采用**双协议并存、统一调度**的架构：

```
主循环 (r_main.c)
  │
  ├─ while (BSP_RS485_Read(&byte, 1))   ← 统一从 UART 缓冲区读字节
  │   ├─ Protocol_ProcessByte(byte)      ← AA55 状态机处理（等 AA 55 帧头）
  │   └─ ModbusProtocol_ProcessByte(byte) ← Modbus 累积+超时检测
  │
  ├─ Protocol_Task()        ← AA55 超时复位（50ms 无字节则重置状态机）
  ├─ ModbusProtocol_Task()  ← Modbus 帧间超时触发（5ms 空闲 → 处理帧）
  └─ R_WDT_Restart()        ← 喂狗
```

**为什么同时喂给两个协议？** 因为 AA55 帧以 `AA 55` 开头，Modbus 帧以从机地址（如 `0x01`）开头，两者不冲突：
- AA55 Parser 遇到非 `AA` 开头的数据待在 `WAIT_AA` 状态，超时后重置
- Modbus Parser 遇到 AA55 帧后会因 CRC 不匹配或地址不匹配而静默丢弃

无需上层通知切换协议，发送什么协议的命令 MCU 就自动识别。

## 固件支持的功能

### AA55 协议命令（`protocol.c`）

| 命令码 | 命令 | 说明 |
|--------|------|------|
| 0x01 | PING | 连接测试，无数据区，返回空 |
| 0x02 | GET_VERSION | 获取版本号，返回 3 字节："1.0" |
| 0x10 | ECHO | 回显测试，原样返回收到的数据 |
| 0x20 | LED_SET | 控制 LED：数据=0x00 开（P77拉低），0x01 关（P77拉高） |
| 0x21 | GET_STATUS | 读取状态：返回 LED 状态（0=开, 1=关） |

### Modbus RTU 功能码（`modbus_protocol.c`）

| 功能码 | 功能 | 寄存器/线圈映射 |
|--------|------|----------------|
| 0x01 | Read Coils | Coil 0 = P77 LED（0=低电平/亮, 1=高电平/灭），其他地址返回0 |
| 0x02 | Read Discrete Inputs | 全部返回 0（预留） |
| 0x03 | Read Holding Registers | Reg 0x0000=主版本号1, 0x0001=次版本号0, 0x0002=LED状态，其他返回0 |
| 0x04 | Read Input Registers | 全部返回 0（预留） |
| 0x05 | Write Single Coil | Coil 0：0xFF00=LED ON, 0x0000=LED OFF |
| 0x06 | Write Single Register | Reg 0x0002：写入 0=LED ON, 1=LED OFF |
| 0x08 | Diagnostics | 子功能 0x0000：返回查询数据（用作 Modbus echo） |

## 上位机界面功能

主窗口 **5 个标签页**：

### 1. 实时监控（Dashboard）
- 仪表盘卡片：母线电压、环境温度、设定温度、风机转速、故障码、LED状态、系统使能
- 实时温度曲线（pyqtgraph）：环境温度（绿线）+ 设定温度（红线）
- 支持 CSV 数据记录（工具栏开关）

### 2. 执行器控制（Actuator Control）
- **Modbus 开关量**：复选框控制 系统使能(Coil 0)、风机使能(Coil 2)
- **AA55 调试**：Ping 测试连接、获取版本号、LED 开关切换
- **执行器参数**：风机 PWM 写入（Holding Reg 3）

### 3. 参数 / EEPROM（Parameters）
- 可编辑的寄存器表格：显示 label/地址/当前值/单位，可直接修改并写入
- "保存参数到 EEPROM" 按钮

### 4. 手动读写（Manual）
- 协议无关的命令界面：下拉选择功能（PING/GET_VERSION/ECHO/GET_STATUS + 所有 Modbus 功能码）
- 手动输入地址、数量、数据，执行后显示 JSON 结果

### 5. 通信日志（Log）
- 实时 TX/RX 十六进制日志（自动滚动）
- 统计信息：TX 帧数 / RX 帧数 / 错误数 / 轮询次数
- 支持日志导出

### 连接栏
- 协议选择：AA55 协议 / Modbus RTU
- 串口扫描、波特率（9600~115200）、校验位、设备地址、轮询间隔
- 自动轮询开关

## 关键技术决策和约定

1. **LED 逻辑**：`0x00 = ON`（低电平点亮），`0x01 = OFF`。AA55 和 Modbus 统一遵守此约定。
2. **寄存器分块读取**：`register_map.py` 的 `spans()` 将地址不连续的寄存器按连续块分组，避免一次读取 100 字节的空数据。
3. **双协议并存**：固件主循环同时喂字节给两个协议解析器，各自识别自己的帧，无需上层通知切换。
4. **CRC16/MODBUS**：多项式 `0xA001`，初始值 `0xFFFF`，小端序输出。固件和 Python 端实现完全一致。
5. **连接容错**：连接时 `get_version()` 失败不阻断连接（version=None），串口打开失败会恢复按钮状态。
